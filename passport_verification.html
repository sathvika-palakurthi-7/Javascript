2. Basic Switch Statement Syntax

switch (expression) { case value1: // code if expression === value1 break; case value2: // code if expression === value2 break; default: // code if no cases match }

3. Switch Fall-through (without break)

When you don't use break, execution continues to the next case:

switch (day) { case 'Monday': case 'Tuesday': case 'Wednesday': case 'Thursday': case 'Friday': console.log('Weekday'); break; case 'Saturday': case 'Sunday': console.log('Weekend'); break; }

7. Switch vs If-Else

When to use Switch:
  • Multiple conditions on a single variable
  • Cleaner and more readable than many if-else statements
  • Better performance for many cases

9. Best Practices for Switch Statements

‚úì DO:
  • Always use break unless intentional fall-through
  • Include a default case to handle unexpected values
  • Group related cases together
  • Keep case logic simple; use functions for complex operations
‚úó DON'T:
  • Forget the break statement (unless intentional)
  • Omit the default case
  • Use complex expressions as cases
  • Use switch for range checking (use if-else instead)
